## 2. 기술 스택 평가

### 2-1. 선택 기술의 적합성
- **Vite 8.x**: 개인 이력서 사이트에 최적화된 경량 빌드 도구. HMR 지원으로 개발 생산성 높음
- **Vanilla JS + CSS**: 프레임워크 의존 없이 구현 — 유지보수 용이, 번들 크기 최소화
- **@napi-rs/canvas**: 별도 시스템 의존성 없이 이미지 생성 가능한 적절한 선택

### 2-2. 빌드 산출물 분석

| 파일 | 크기 | 평가 |
|------|------|------|
| `index.html` | 7.73 kB | 적정 |
| CSS | 6.44 kB (gzip 1.94 kB) | 우수 |
| JS | 3.10 kB (gzip 1.35 kB) | 우수 |
| 프로필 사진 | 418.49 kB | 개선 여지 있음 (WebP 변환 시 ~60% 절감 가능) |

**총 초기 로딩 리소스:** 약 430 kB (사진 제외 시 약 11 kB)

## 3. 디자인 시스템 평가

### 3-1. 컬러 시스템 완성도 ✅
```
주컬러    : #4169E1 (Royal Blue)         → 섹션 제목, 강조 요소
보조컬러  : rgba(65,105,225,0.5)          → 태그, 보조 테두리
이벤트컬러: #f472b6 (Light Pink)          → 호버 강조, 수상 포인트
Hero 전용 : #0D2D84                        → 딥 네이비 배경
```
3색 체계가 일관되게 적용되어 시각적 위계(hierarchy)가 명확함

### 3-2. 다크모드 구현 ✅
- OS 설정 연동 자동 전환 (`prefers-color-scheme`)
- 라이트/다크 각각 6개 CSS 변수로 완전 분리
- 주컬러 다크모드 조정(`#7b9ff5`) — 어두운 배경 대비비 8.2:1 확보

### 3-3. 반응형 디자인 ✅
- 640px 기준 모바일 레이아웃 분기
- 모바일: 세로 스택 레이아웃, 중앙 정렬
- 타임라인, 프로젝트 그리드 1열 전환
